---------------------------------------------------------------------------------------------------
Version: 0.7.3
Date: 2026-05-19
  Bugfixes:
    - Fixed on_tick crash: LuaTechnologyPrototype does not expose custom data-stage fields in
      Factorio 2.x. Removed the technology.castra_prime_ignore mod-author API; the recipe
      equivalent (recipe.castra_prime_ignore for the Forge) is unaffected.
    - Fixed Forge entity description incorrectly advertising a quality bonus that was removed
      in this version IF buffed forge setting is on
    - Fixed mining a player-crafted jammed data collector returning the internal
      jammer-interference fuel item to the player's inventory.
  Features:
    - Added "Forge glow" setting (on by default): enables the light emission layer on the Forge.
    - Added "Extend data packet spoilage time" setting: increases battlefield data packet
      spoilage from 1 minute to 5 minutes.
    - Player-crafted jammed data collectors are now minable and deconstructable. Jammer-converted
      enemy data collectors remain permanent as before.
  Balance changes:
    - Buffed Forge setting: removed the 10% quality bonus (productivity bonus unchanged at +50%).
    - Buffed Forge setting: tanks and spider vehicles can now be crafted in the Forge.

Version: 0.7.2
Date: 2026-05-19
  Bugfixes:
    - Fixed periodic lag spikes (~every 33 seconds) caused by stale data collector references
      accumulating in storage when entities were destroyed during quality upgrades without
      raising the destroy event. Also fixed a hard freeze that could occur if all data
      collectors were destroyed while stale references remained in the list.
    - Fixed mod-name locale key (castra -> castra-prime) so the planet symbol displays
      correctly in the mod list.
---------------------------------------------------------------------------------------------------
Version: 0.7.1
Date: 2026-05-14
  Bugfixes:
    - Fixed personal laser defenses not targeting enemy carbon fiber walls; the prototype was
      missing is_military_target = true
    - Fixed personal laser defenses not targeting enemy roboports, solar panels, and power poles;
      these are now placed as castra-enemy-* prototype variants with is_military_target = true.
      Manual targeting was unaffected — only automatic weapons were skipping these structures.
    - Fixed disable-enemy-nukes not applying to artillery turrets; nuclear artillery shells
      (cerys-neutron-bomb, maraxsis-fat-man) are now capped to standard artillery-shell when
      the setting is enabled.
    - Added localization for castra-enemy-* infrastructure prototypes (roboport, solar panel,
      power poles) — these now display as "SIMULAC [Entity]" instead of an unknown key error.
  Known incompatibilities:
    - Igrys: that mod's mutually exclusive tech paths cause Castra's enemy AI to bounce
      between the two techs indefinitely. No fix planned.

Version: 0.7.0
Date: 2026-05-13
  Fixed issue with starting area resource gen? 
  Bugfixes:
    - Fixed crash when placing a jammer radar while jammed data collectors already existed in
      storage; jammed_data_collectors_jammers table was not initialized in the jammer built handler
    - Fixed pollution cache variable typo that wiped data collector pollution readings on every
      collector destroyed, causing data drop rates to be permanently suppressed
    - Fixed turret upgrade filtering: enemy turrets were always skipped due to checking the wrong
      table, so upgrades never actually placed new turrets
    - Fixed productivity module tier: enemy research speed bonus for productivity modules was
      reading the speed module tier instead of the productivity module tier
    - Fixed find_closest_entity: distance was calculated from the world origin instead of the
      reference position, causing power poles to connect to wrong entities
    - Fixed place_turrets: a double-remove bug could skip entities when filtering turret lists

  Balance changes:
    - Removed quality-module-2 as a prerequisite for the forge technology. The built-in quality
      bonus on the forge made requiring quality research a soft circular dependency, and blocked
      the craft-item research trigger entirely for players without quality techs unlocked.
    - Added option to prevent enemy rocket turrets from loading nuclear ammo (atomic/hydrogen
      bombs). Existing nuclear ammo in turrets will remain until fired.
    - Replaced "Include Battlefield in Research Prod" setting with "Castra Gates Late-Game
      Progression" (on by default). When disabled, lithium batteries are no longer required
      for railgun and promethium science pack recipes, and battlefield science is not required
      for promethium science or research productivity research.
    - Fixed: lithium battery was always required in the promethium-science-pack recipe even
      when the old include-promsci setting was off.

  Ease of use:
    - Battlefield data packets (castra-data) can now be requested through the logistic network
      and set as logistic filters. Previously the item was invisible to filter UIs because it
      has no unlocked producing recipe.
    - Added option to remove battlefield science pack requirement from atomic bomb technology,
      allowing nuclear weapons to be researched without visiting Castra.
    - Fixed recycling recipes for battery-mk3-equipment, railgun, and railgun-turret to return
      lithium batteries, matching the ingredient changes made by this mod.
    - Data collectors now spawn items and tanks in peaceful mode. The planet is intended to be
      a combat challenge regardless of map settings.
    - Mod authors can set recipe.castra_prime_ignore = true to prevent Castra Prime from
      re-categorizing their recipes for the Forge.

  Code:
    - Extracted shared constants (radii, turret list, hyphen_to_underscore) to castra-config.lua
    - Extracted convex hull and wall geometry math to wall-geometry.lua
    - Added documented TODOs for two known architectural issues in base-upgrades.lua
